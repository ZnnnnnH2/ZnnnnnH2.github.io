# ZnnnnnH2.github.io

This repository is the GitHub Pages deployment repository for **ZnnnnnH2 小屋**.

Public site:

- <https://ZnnnnnH2.github.io>

## Repository role

This repository currently contains generated static site output, not the clean source project.

Typical generated paths include:

- `index.html` — homepage generated by the static site generator.
- `2025/.../index.html` — generated post pages.
- `archives/` — generated archive pages.
- `categories/` — generated category index pages.
- `tags/` — generated tag index pages.
- `css/`, `js/`, `img/` — generated/static assets used by the site.
- `sitemap.xml`, `baidusitemap.xml` — generated search-engine sitemap files.

## Maintenance rules

Do not manually reorganize generated folders such as `2025/`, `archives/`, `categories/`, or `tags/` unless you are intentionally changing the deployed output. Moving those files will break existing URLs.

For normal blog maintenance, edit the source-site project, then regenerate and redeploy this repository.

## Domain status

The expired custom domain `znnnnnh2.icu` has been removed from the GitHub Pages `CNAME` configuration. The active default Pages URL is:

- <https://ZnnnnnH2.github.io>

Some generated HTML metadata may still contain old absolute URLs until the site is regenerated with the new `url` setting in the source-site configuration.

Recommended source-side fix:

```yaml
url: https://ZnnnnnH2.github.io
root: /
```

Then regenerate and redeploy the site.

## Suggested clean workflow

1. Keep this repository as the deployment target only.
2. Keep Markdown posts, theme config, and generator config in a separate source repository.
3. Regenerate the static site from source instead of editing generated HTML directly.
4. Deploy generated output into this repository.
