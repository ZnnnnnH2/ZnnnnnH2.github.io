反序列化

类型 标识符 格式 例子

class

php 中对 class 的修饰

  • private：修饰成为\0ClassName\0propertyName格式
  • protected：修饰成为\0*\0propertyName格式
  • public：不做修饰

注：\0为 ascii 码中的 null 字符，urlencode 后为 %00，php 中表示为 chr(0)

魔术方法

  • __construct()：对象创建时调用
  • __destruct()：对象销毁时调用
  • __wakeup()：反序列化时调用
  • __sleep()：序列化时调用
  • __toString()：对象当作字符串使用时调用
  • __invoke()：对象当作函数使用时调用
  • __clone()：对象克隆时调用
  • __call()：调用未定义或不可见的方法时调用
  • __callStatic()：调用未定义或不可见的静态方法时调用
  • __get()：访问未定义或不可见的属性时调用
  • __set()：给未定义或不可见的属性赋值时调用
  • __isset()：调用 isset() 或 empty() 时调用
  • __unset()：调用 unset() 时调用

细节

wakeup 绕过

CVE-2016-7124：如果存在__wakeup方法，调用 unserialize()方法前则先调用__wakeup()方法，但是序列化字符串中表示对象属性个数的值大于真实的属性个数时会跳过__wakeup()的执行

漏洞影响版本：PHP5 < 5.6.25 PHP7 < 7.0.10

例题

MRCTF2020 Ezpop

突破点为类Modifier中的append方法，利用include($value);进行文件包含。而append方法在__invoke魔术方法中被调用，所以需要构造一个Modifier对象，并设置其受保护属性var为想要包含的文件路径。

在Test类中，__get魔术方法会调用属性p，而p被初始化为一个空数组。通过构造一个Test对象，并设置其属性p为一个闭包函数，该函数返回一个Modifier对象，我们可以间接调用Modifier的__invoke方法，从而触发文件包含。

在Show类中，__toString魔术方法返回属性str的source属性，只要将str设置为一个Test对象，就可以通过字符串转换触发Test的__get方法，进而调用Modifier的__invoke方法，实现文件包含。

而Show类的__wakeup方法中有一个正则表达式过滤，将其source属性设置为Show类，则会自动调用__toString方法，从而触发链式调用。
